Main Concepts Router - And More

  • Essentials Router features are now covered
  • But Router is a complete library which answers to most routing problematics
  • For example, you've got hooks to disable the routing of a component conditionally, or not authorize to acceed to a component:
    • CanActivate: implement the CanActivate interface by adding the routerCanActiate method, and return either a boolean or a Promise to authorize the component to be activated or not.
    • CanDeactivate: CanActivate allows to authorize or not to access to a component view, whereas CanDeactivate authorize or not to leave a component view
    • OnActivate: the routerOnActivate method is called when the routing is successuffly done, if the function returns a Promise, the router will wait it is solved bedore instanciating child components
    • OnDeactivate: the routeOnDeactivate is called before the leaved component is destroyed, if the function returns a Promise, the router will wait it is solved before destroying the view
