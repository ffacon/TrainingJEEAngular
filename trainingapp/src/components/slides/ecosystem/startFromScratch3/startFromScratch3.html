Ecosystem Start From Scratch - Install angular-cli

  • Angular provides a utility called angular-cli to allow users to create and manage projects from the command line.
    It automates tasks with the following commands:
    • ng new : Creates projects already configured with Webpack as a build tool.
    • ng serve : Builds and serves your app, rebuilding on file changes.
    • ng generates : Generates and/or modifies files based on a schematic.
    • ng build : Compiles an Angular app into an output directory named dist/ at the given output path.
    • ng e2e : Builds and serves an Angular app, then runs end-to-end tests using Protractor.
    • ng test : Runs unit tests in a project.

    We recommend use of angular-cli as it will help create and maintain common patterns across our application.

  • To install latest version of angular-cli globally, just run the following command:
    $ npm install -g @angular/cli

    Since v6, major version of angular-cli follow the supported major version of angular.
    So if you want to create a new project that use angular V8.2.x, you will have update the angular-cli version to the latest released version on the 8.2.x branch.

    To apply the update locally, just run the following command:
    $ npm install @angular/cli@8.2.2
