Main Concepts About Modules - Modules dependencies

  • When you will work on big applications, you will want to split your code
  • Split your code is already done by using single files for Component, Services, Pipes and so on
  • But what if you want to create such kind of reusable libraries? For that, you will need to create custom modules that can be imported in other modules
  • By default, the Component/Directives/Pipes that you will create in your module are not available for other ones (if module A imports module B, by default, A can't access to custom B's components)
  • To make a Component/Directives/Pipes available, you have to use the exports field @Component({ selector:'cmp2', template:`{{'Sub component' | customPipe}}` }) class Cmp2{} @Component({ selector:'usefull-cmp', template:`...` }) class UsefullComponent{} @NgModule({ imports: [BrowserModule], declarations: [UsefullComponent], //usable in the module itsel exports:[UsefullComponent, BrowserModule] // usable in module which import BrowserModule }) export class LibModule { }
  • In the example below, an other module is exported from LibModule, it means that a module which imports LibModule won't needd to import BrowserModule manually
