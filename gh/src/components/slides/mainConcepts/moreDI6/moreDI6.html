Main Concepts Dependency Injection - More Details - Usage

All previous notions can be used in your apps, but generally you will not need to use the injector object. The declaration of providers will be deported:

  • To the module declaration (@NgModule)
  • To a component declaration (@Component)
  • Remember the tree of injectors we have seen previously
    • Declare a provider on a component make it available for its children components only
    • By default, a service is a singleton
    • But if a service is provided twice, then let's have one instance by provider
import {NgModule, Component} from '@angular/core'; import {BrowserModule} from '@angular/platform-browser'; import {CustomService} from './custom-service'; @Component({ selector: 'app', template: 'Hello World!' }) class AppCmp{} @NgModule({ imports: [BrowserModule], declarations: [AppCmp], providers:[ CustomService ], bootstrap:[ AppCmp ] }) export class AppModule{} import {NgModule, Component} from '@angular/core'; import {BrowserModule} from '@angular/platform-browser'; import {CustomService} from './custom-service'; @Component({ selector: 'app', template: 'Hello World!', providers: [ CustomService ] }) class AppCmp{} @NgModule({ imports: [BrowserModule], declarations: [AppCmp], bootstrap:[ AppCmp ] }) export class AppModule{}
