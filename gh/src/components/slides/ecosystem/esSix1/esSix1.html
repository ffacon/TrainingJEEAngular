Ecosystem ES6 Introduction - var & let

Hoisting: var allows in js to declare a variable, but it has a specific behavior which can be surprising and error prone, wherever your variable is declared, it is accessible to the whole function. The two versions below (var1.js and var2.js) are equivalent.

function checkScope(i){ if ( i> 0){ var msg= "greater than zero"; } else{ var msg= "less or equal to zero"; } console.log(msg); } checkScope(0); checkScope(1); function checkScope(i){ var msg; if ( i> 0){ msg= "greater than zero"; } else{ msg= "less or equal to zero"; } console.log(msg); } checkScope(0); checkScope(1);

With let, the variable scope is the block where it is defined, as we are used to in much languages. Prefer using let than var
