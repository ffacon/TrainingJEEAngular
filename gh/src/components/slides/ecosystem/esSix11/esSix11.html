Ecosystem ES6 Introduction - Modules

The modules are both a way to answer to the hot loading problematic and to help organizing the code. Previously, there were several solutions/syntaxes to this problem, whose most famous are:

  • CommonJs: Which is used in nodeJs for example
  • AMD: With requireJs for example, gives a way to define several modules in one file and is more targeted for browser applications because modules are loaded asynchronously

But both of them answer to most problematics and are fine, they also are compatible with the ES5 syntax

exports.add = function() { var sum = 0, i = 0, args = arguments, l = args.length; while (i < l) { sum += args[i++]; } return sum; }; var add = require('common-math').add; exports.increment = function(val) { return add(val, 1); }; define(['math'], function (math) { return { add: function(){ var sum = 0, i = 0, args = arguments, l = args.length; while (i < l) { sum += args[i++]; } return sum; } }; }); define(['increment'], ['math'], function (math) { return { increment: function(val){ math.add(val, 1); } }; });
