Advanced Concepts Unit Tests - Testing pipes

Finally, we have to write our tests

  • First question, Where? Basically where you want, but put the unit tests files just next to the files to test makes sense ( easier to find, remember to update, move them together... )
  • Below is a first example to test a pipe
  • The main method is not a jasmine/karma constraint, it's our own convention
import {Pipe, PipeTransform} from '@angular/core'; @Pipe({name:'identity'}) export class IdentityPipe implements PipeTransform{ constructor(){} transform(input: any, args: any){ return input; } } import {IdentityPipe} from './identityPipe'; export function main() { describe('UpdateDataPipe', function() { it('Update data', function() { let pipe = new IdentityPipe(); let data: string = 'data content'; let result: string= pipe.transform(data, []); expect(result).toEqual(data); }); }); }
